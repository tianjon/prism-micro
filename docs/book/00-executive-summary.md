# 执行摘要

---

## 一句话定位

**Prism 是一个 AI Native 的客户声音（VOC）分析平台，让组织的每一条客户声音都被理解、被连接、被行动。**

---

## 市场痛点

企业每天收到数以千计的客户反馈——App 评论、客服对话、社交媒体、NPS 调查——但传统 VOC 工具只能做到三件事：分类、计数、出报表。当管理者问"我们为什么总是慢半拍"时，仪表板上的环比箭头没有答案。

问题不在数据量不够，而在于**三个系统性失败**：

1. **分类陷阱**：预设的标签体系永远追不上用户表达的多样性，30% 的反馈落入"其他"——恰恰是最值得关注的新兴问题。
2. **关键词幻觉**：用户说"像 PPT 一样"来描述卡顿，关键词系统完全无法捕获——传统方法遗漏近 70% 的相关反馈。
3. **仪表板幻觉**：NPS 从 4.2 涨到 4.3，数字精确但分类不精确，真正重要的新问题被淹没在大类中。

当企业试图用 AI "解决"这些问题——让 LLM 自动分类、自动写周报——结果往往是：**结论更快、成本更高、信任更低**。AI 只是给旧引擎加了涡轮增压，但方向盘还是歪的。

---

## 我们的方案

Prism 不是给传统 VOC 换一个更聪明的引擎，而是**重新定义了 VOC 的系统语言**：

1. **涌现式标签取代预设分类**：AI 从数据中自动发现概念，而非把反馈塞进事先画好的格子。语义覆盖率从 30% 提升到 85%+。
2. **Signal → Concept 两阶段知识资产管理**：信号是矿石，概念是精炼后的金属。AI 负责发现与提出，人类负责确认与运营，组织知识越用越强。
3. **Agent-First 双用户架构**：系统同时为人类和 AI Agent 设计。Agent 是不知疲倦的研究员，在海量语义中探索、联想、验证；人类是编辑与决策者，把发现沉淀为组织资产。

---

## 核心差异化

- **AI Native，不是 AI-Enhanced**：向量空间优先、原子工具优先、涌现优先于预设——从底层重建，而非在旧系统上叠加。
- **Agent 是一等公民**：所有 API 从第一天就考虑"人和 Agent 都是调用者"，系统天然具备被 AI 编排的能力。
- **知识复利结构**：每确认一个概念，未来的新反馈就更容易归类与解释——误报降低、行动加速、创新更敏捷。

## 路线图概览

| 阶段 | 内容 | 核心交付 |
|------|------|---------|
| Phase 1 | 基础设施 + 模型配置 | 微服务骨架、LLM Provider 管理 |
| Phase 2 | LLM 调用能力 | 统一 Chat/Embedding/Rerank API、故障转移 |
| Phase 2.5 | Agent 基础运行时 | Skill 注册表、基础 Agent Loop、双身份认证 |
| Phase 3 | VOC 数据摄入 + 语义底座 | 四阶段 AI 管线、涌现式标签、向量检索 |
| Phase 4 | 语义检索 + 概念治理 | Concept Layer、人机共治工作台 |
| Phase 5 | Agent 高级交互 + 编排 | 多 Agent 协作、自定义 Skill、工作流编排 |
| Phase 6 | 洞察引擎 + 平台化 | 自动化洞察发现、开放平台 |

每个阶段独立可交付、可演示，并设有 Go/No-Go 评审点。

## 资源需求与预期回报

**核心团队**：后端 ×2 + 前端 ×1 + AI 工程师 ×1 + 产品经理 ×0.5（约 4.5 人）

**时间线**：Phase 1-2 已在进行中；Phase 2.5-4 约 18-24 周可达到核心功能可用。

**投资回报**：
- 替代 1-2 名全职 VOC 分析师的重复性工作（年薪 30-50 万）
- 问题检测时间从"周级"缩短到"天级"，减少客户流失风险
- 对标 Qualtrics/Medallia 年费 50-200 万，自建方案长期成本优势显著
